# 第 3 月学习测验：HTTP/HTTPS 请求复现

## 理论题
1. Header、Cookie、Body 各自的作用是什么？
2. Session 和 Token 的常见区别是什么？
3. 为什么浏览器里成功的请求，脚本中可能失败？
4. 哪些情况下需要重点检查重定向、超时和代理？

## 实操题
1. 写一个 GET 请求脚本并解析返回结果。
2. 写一个 POST JSON 请求脚本。
3. 写一个 Session 登录脚本并访问登录后资源。
4. 写一个文件上传脚本。
5. 根据抓包信息还原一个完整请求脚本。

## 通过标准
- 能独立完成请求复现
- 能分析请求失败是 Header、Cookie、参数还是编码问题
- 能将常用请求整理为可重复执行脚本
