# Python 网络编程与逆向辅助 6 个月学习测验

> 使用方法：每个月结束后做一次测验。先做理论题，再做实操题，最后根据通过标准判断是否进入下个月。

---

## 第 1 月测验：Python 基础与脚本化

### 理论题
1. `list`、`tuple`、`set`、`dict` 分别适合存什么数据？
2. `dict.get()` 和 `dict[key]` 有什么区别？
3. `try/except/finally` 各自作用是什么？
4. `if __name__ == '__main__':` 有什么意义？

### 实操题
1. 写一个脚本，统计文本中每个单词出现次数。
2. 写一个脚本，读取 JSON 文件并提取指定字段。
3. 写一个脚本，接收命令行参数后执行不同逻辑。

### 通过标准
- 能独立写 100 行以内脚本
- 能熟练操作字符串、列表、字典、文件、JSON
- 能处理基础异常和命令行参数

---

## 第 2 月测验：数据处理与编码解码

### 理论题
1. `bytes` 和 `str` 的区别是什么？
2. Base64、Hex、URL 编码分别常见于什么场景？
3. MD5、SHA256、HMAC 的区别是什么？
4. 为什么响应内容有时需要先解压再解析？

### 实操题
1. 写一个编码解码工具，支持 Base64、Hex、URL。
2. 写一个脚本，输入消息和密钥后输出 HMAC-SHA256。
3. 写一个脚本，对 gzip 或 zlib 数据进行解压。
4. 写一个 JSON 字段提取工具。

### 通过标准
- 能判断数据当前是文本、字节、编码串还是压缩内容
- 能写出常见编码解码和哈希小工具
- 能解释简单签名值的生成思路

---

## 第 3 月测验：HTTP/HTTPS 请求复现

### 理论题
1. Header、Cookie、Body 分别起什么作用？
2. Session 和 Token 的常见区别是什么？
3. 为什么浏览器请求成功，脚本请求可能失败？
4. 什么情况下要重点检查重定向、超时和代理？

### 实操题
1. 写一个 GET 请求脚本并解析响应。
2. 写一个 POST JSON 请求脚本。
3. 写一个 Session 登录脚本并访问登录后页面。
4. 写一个文件上传脚本。
5. 根据抓包信息还原一个完整请求脚本。

### 通过标准
- 能稳定复现普通接口请求
- 能定位请求失败是 Header、Cookie、参数还是编码问题
- 能将一组请求整理成可重复执行的 Python 脚本

---

## 第 4 月测验：Socket 与简单协议

### 理论题
1. TCP 和 UDP 的主要区别是什么？
2. 什么是粘包和拆包？
3. 为什么协议中经常会有长度字段？
4. `struct.pack()` / `struct.unpack()` 是做什么的？

### 实操题
1. 写一个 TCP 客户端，向服务端发送消息。
2. 写一个 TCP 服务端，接收消息并回显。
3. 写一个 UDP 消息发送和接收程序。
4. 自定义“长度 + 类型 + 数据”报文并完成打包/解包。

### 通过标准
- 能完成基础 TCP/UDP 通信实验
- 能读懂简单二进制报文结构
- 能解释报文如何从字节流还原为字段

---

## 第 5 月测验：异步编程与 WebSocket

### 理论题
1. 协程和线程的常见区别是什么？
2. 为什么异步更适合批量网络请求？
3. 什么是并发数限制？
4. WebSocket 和普通 HTTP 请求有什么主要差别？

### 实操题
1. 写一个异步批量请求器，请求多个 URL。
2. 为异步请求器加入超时和重试逻辑。
3. 写一个并发下载器，统计成功数和失败数。
4. 写一个 WebSocket 客户端，打印服务器消息。

### 通过标准
- 能写异步网络工具
- 能控制并发、超时、重试和日志
- 能连接并观察 WebSocket 消息流

---

## 第 6 月测验：抓包分析与综合实战

### 理论题
1. 动态参数一般如何识别？
2. 为什么分析请求时要关注请求顺序？
3. 如何判断一个字段可能参与签名？
4. 为什么需要多次对比实验？

### 实操题
1. 对同一接口抓取多次请求，标注固定字段和变化字段。
2. 写一个参数差异对比脚本。
3. 写一个签名验证脚本，验证某字段是否由特定规则生成。
4. 写一个响应解码/解压脚本。
5. 输出 1 份完整案例文档：抓包、分析、验证、复现。

### 通过标准
- 能独立完成一个完整请求链路分析案例
- 能用脚本验证参数和签名逻辑
- 能形成结构化分析笔记与复盘文档

---

## 6 个月结业总测验

### 理论总测验
1. 描述一次 HTTP 请求从构造到响应解析的完整过程。
2. 说明 Cookie、Session、Token 的作用差异。
3. 说明 `bytes`、编码、压缩、哈希在网络分析中的关系。
4. 说明 TCP、UDP、WebSocket 的适用场景。
5. 说明异步并发为什么适合批量网络任务。
6. 说明动态参数分析的一般步骤。

### 综合实操总测验
1. 从一段抓包记录中提取关键请求。
2. 使用 Python 复现该请求。
3. 识别关键 Header / Cookie / Body 字段。
4. 对变化字段做差异分析。
5. 编写验证脚本确认某个参数逻辑。
6. 输出完整分析文档。

### 结业标准
达到以下 4 条即可认为完成路线：
- 能写：会写稳定实用脚本
- 能看：看懂请求、响应、字节和编码
- 能复现：能将请求复现成脚本
- 能验证：能用脚本验证参数和逻辑
